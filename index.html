  • Traditional MLS: Requires real estate license and MLS membership (most comprehensive data)
  • Alternative Sources: No licensing required, varying costs and coverage (see options below)

For MLS access, contact your local Multiple Listing Service. For alternatives, sign up directly with the data providers listed below.

Step 2: Get API Credentials

Depending on your chosen option:

  • MLS APIs: Requires approval, membership, and licensing
  • Alternative APIs: Sign up directly with provider
  • Public Records: Often no registration required
  • Scraping APIs: Sign up for free tier or paid plans

Step 3: Configure Your Application

Add your API credentials to the DATA_CONFIG section:

// Example: Traditional MLS Configuration const DATA_CONFIG = { mls: { bridge: { apiKey: 'YOUR_BRIDGE_API_KEY_HERE', enabled: true // Set to true when configured } } }; // Example: Alternative Data Sources const DATA_CONFIG = { alternative: { rentals: { rentalsApi: { apiKey: 'YOUR_RENTALS_API_KEY', enabled: true } }, scraping: { scrapingBee: { apiKey: 'YOUR_SCRAPINGBEE_KEY', enabled: true } }, publicRecords: { enabled: true // No API key needed for public data } } };

Step 4: Verify Your Connection

After saving your credentials:

  • The status banner will automatically update to show your connection status
  • Your MLS number will be displayed in the banner for confirmation
  • Search results will immediately show real property listings
  • Use the "Test Connection" button to verify your API access
  • Close this modal to start searching for properties
Summary: You have multiple options for real estate data:
  • Traditional MLS: Most comprehensive, requires real estate license ($$$)
  • Alternative APIs: Good coverage, some free tiers ($ - $$)
  • Public Records: Historical data only, completely free ($)
  • Web Scraping: Real-time but requires legal compliance ($ - $$)
Choose based on your budget, data needs, and compliance requirements.
